# Лабораторная работа №3: классификация и прогнозирование временных рядов метеоданных

Проект реализует двухэтапную систему анализа временных рядов для метеоданных шести городов России:

1. **Классификация временного ряда** по фиксированному окну наблюдений: определение климатического типа / города.
2. **Прогнозирование температуры на 30 дней вперёд**: отдельная прогнозная модель для каждого города.

Основной акцент сделан на специализированные модели для временных рядов, а не на бустинги:

- классификация: `InceptionTime`-подобная 1D CNN для multivariate time series classification;
- прогнозирование: `TCN` (Temporal Convolutional Network) с каузальными dilated convolution blocks;
- опционально в ноутбуке можно переключить классификационную цель с `climate_zone` на `city`.

## Что делает ноутбук

- Загружает parquet-файлы за 2019–2025 годы по шести городам.
- Проверяет пропуски, выбросы, сезонность, распределения, Q-Q plots, корреляции.
- Выполняет cross-city анализ: среднемесячная температура, годовые осадки, климатические профили городов.
- Проверяет стационарность температуры тестом Дики-Фуллера и показывает результат после дифференцирования.
- Генерирует временные, циклические, лаговые, rolling, динамические и агрегированные признаки.
- Строит PCA-визуализацию разделимости окон.
- Обучает InceptionTime-like классификатор временных рядов.
- Считает Accuracy, Precision, Recall, F1 macro/weighted, ROC-AUC One-vs-Rest, confusion matrix.
- Анализирует ошибки по месяцам и сезонам, а также permutation importance по каналам временного ряда.
- Обучает отдельные TCN-модели прогнозирования температуры для каждого города.
- Считает MAE, RMSE, MAPE, WAPE по каждому горизонту 1–30 дней.
- Выполняет анализ остатков: Shapiro-Wilk, Ljung-Box, White test.
- Объединяет классификатор и прогнозные модели в единый pipeline-класс.

## Интерпретация результатов

- Если `climate_zone` классифицируется лучше, чем `city`, это ожидаемо: пары городов внутри одной климатической зоны имеют похожие сезонные профили.
- Ошибки классификации чаще возникают на межсезонье, где температурные и влажностные режимы разных регионов сближаются.
- Для прогноза на 30 дней ошибка обычно растёт с горизонтом, потому что атмосферные процессы хаотичны, а дальний прогноз сильнее опирается на сезонную норму, чем на точное текущее состояние.
- TCN выбран как компромиссная модель: он учитывает локальные паттерны, сезонную структуру и долговременные зависимости через dilated convolutions, но обучается быстрее рекуррентных seq2seq-моделей.

## Почему не бустинги

Бустинги можно использовать как табличный baseline на handcrafted features, но в этой работе они не являются основной моделью, потому что преподаватель отдельно просила использовать специализированные модели классификации временных рядов. Поэтому основной классификатор работает напрямую с последовательностью `window × features`, а не только с плоской табличной агрегацией.
