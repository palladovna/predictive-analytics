# Лабораторная работа №2
## Виртуальный датчик для контроля процесса обжига в печи

Репозиторий содержит решение лабораторной работы по предиктивной аналитике для задачи построения виртуального датчика, прогнозирующего текущее значение концентрации выходного продукта по минутной телеметрии печи и нерегулярным лабораторным замерам.

## Состав репозитория

- `script.ipynb` — основной Jupyter Notebook с полным пайплайном решения
- `data_train.csv` — минутная телеметрия для обучения
- `target_train.csv` — лабораторные замеры для обучения
- `data_test_small.csv` — минутная телеметрия для тестирования
- `target_test_small.csv` — лабораторные замеры для тестирования
- `requirements.txt` — список зависимостей
- `README.md` — инструкция по запуску и интерпретации результатов

## Постановка задачи

Необходимо спрогнозировать значение целевого показателя в текущий момент времени с учетом временной динамики процесса обжига.

Особенность задачи состоит в том, что:
- телеметрия доступна каждую минуту;
- лабораторные измерения целевой переменной поступают нерегулярно;
- лабораторный результат приходит с задержкой примерно 10–15 минут;
- оператору требуется оценка показателя в реальном времени, без ожидания лабораторного ответа.

Поэтому задача решается как задача построения виртуального датчика на основе временных рядов.

## Что сделано в решении

В ноутбуке реализованы следующие этапы:

1. Загрузка и первичная проверка данных  
   Выполняется чтение телеметрии и лабораторных замеров, проверка структуры таблиц, типов данных и имен колонок.

2. Разведочный анализ данных  
   Анализируются:
   - тренды и поведение телеметрических признаков;
   - распределение целевой переменной;
   - частота и нерегулярность лабораторных измерений;
   - пропуски и потенциально неинформативные признаки.

3. Синхронизация телеметрии и target  
   Лабораторные замеры сдвигаются назад на предполагаемую задержку отбора/анализа пробы, после чего каждому замеру сопоставляется ближайшее предшествующее состояние телеметрии через `merge_asof`.

4. Инжиниринг признаков  
   Формируются:
   - лаговые признаки;
   - скользящие статистики;
   - признаки изменения динамики (`diff`);
   - компактный набор признаков для статистической модели.

5. Обучение моделей  
   В работе рассматриваются минимум три подхода:
   - `Ridge` — интерпретируемый baseline;
   - `RandomForestRegressor` — нелинейная ансамблевая модель;
   - `SARIMAX` — статистическая модель временных рядов с экзогенными признаками.

6. Оценка качества  
   Рассчитываются метрики:
   - `MAE`
   - `RMSE`
   - `MAPE`
   - `WAPE`
   - `Direction Accuracy`

7. Анализ остатков  
   Проводится анализ распределения остатков, проверка автокорреляции и оценка признаков гетероскедастичности.

8. Интерпретация результатов  
   Выполняется анализ важности признаков и формулируются аналитические выводы по каждому крупному блоку.

## Почему выбраны именно эти модели

### Ridge
Используется как простой и интерпретируемый baseline. Позволяет проверить, насколько хорошо задача решается линейной комбинацией лаговых и агрегированных признаков.

### RandomForestRegressor
Подходит для табличных данных с большим количеством лагов и rolling-признаков. Позволяет моделировать нелинейные зависимости без сложной настройки.

### SARIMAX
Подходит как статистическая модель временного ряда с экзогенными переменными. Дополнительно позволяет использовать информационные критерии AIC/BIC, что прямо соответствует требованиям задания.

## Какие модели не подходят и почему

- `KNN` — плохо масштабируется на больших объемах данных и не учитывает временную структуру напрямую.
- `SVR/SVM` — вычислительно тяжелы для такого объема данных и большого количества временных признаков.
- `LSTM/GRU` — для лабораторной работы являются избыточно сложными, требуют дополнительной подготовки последовательностей и хуже интерпретируются.
- Случайное перемешивание данных перед split недопустимо, так как приводит к утечке будущей информации.

## Структура данных

### Телеметрия
Файлы `data_train.csv` и `data_test_small.csv` содержат:
- колонку времени `datetime`;
- 16 телеметрических признаков: `telemetry_0` ... `telemetry_15`.

### Целевая переменная
Файлы `target_train.csv` и `target_test_small.csv` содержат:
- колонку времени (в исходных данных — `Дата`, далее приводится к `datetime`);
- колонку `target` — лабораторный замер концентрации продукта.

## Интерпретация результатов

При анализе итогов модели важно обращать внимание на следующее:

- **MAE / RMSE** показывают среднюю абсолютную и квадратичную ошибку прогноза.
- **MAPE / WAPE** показывают относительную ошибку, что удобно для сравнения качества в масштабах целевой переменной.
- **Direction Accuracy** показывает, насколько хорошо модель улавливает направление изменения показателя во времени.
- Визуальное сравнение факта и прогноза позволяет понять, насколько модель справляется с резкими переходами и нештатными режимами.
- Анализ остатков помогает определить, не остается ли в ошибках неучтенная временная структура.

Если лучшая модель показывает устойчиво низкую ошибку и адекватно отслеживает динамику `target`, то ее можно рассматривать как основу виртуального датчика для поддержки оператора в реальном времени.

## Итог

В результате работы построен прототип виртуального датчика, который:
- использует минутную телеметрию процесса;
- учитывает задержку лабораторного измерения;
- прогнозирует текущее значение целевого показателя;
- позволяет сократить задержку получения информации о качестве выходного продукта.

Такой подход может использоваться как инструмент поддержки принятия решений оператором при управлении процессом обжига.
